Reunixance

This is not my own creation at all, but rather a set of principles I have a great deal of interest in, due mostly to its promotion of good programming practices through heavy author limitation, and its ideals of greatly simplifying several fields of computing including networking/web through the use of preexisting tools (often Unix utilities or properties of the filesystem) in place of more consuming dedicated software. Another reason may be that it was founded by an exceptionally good friend of mine. At any rate, I leave a link to their website here

Life of Derek

The 'Life of Derek' video game is another project with a multi-faceted development. Originally a text based game written in Python (the majority of the code for which was written by my friend), I later worked on a more personal rendition written very crudely first in GDScript, and then in C#, using digitalised sketches for sprite work. I wrote two songs for the OST as well, and had plans for more, but as ever once the project had reached a natural point of near completion, others beckoned.

NigolOS

NigolOS is a moderately ambitious set of projects to develop a personal operating system. One of these is a GNU/Linux distribution compiled from scratch and made reproducible for the x86-64 architecture. Another was a 64 bit kernel written using C and Assembly, which currently only has basic stdout. This operating system is a preferable end goal compared to the GNU/Linux distribution, but the former was a useful learning experience with regards to userland utilities. My aim is to have a kernel with fully functional I/O, so that Nigolas may receive as well as give..

Nigolos

Nigolos is a very simple TUI 'window manager' written in C using NCurses.
The idea was to have an extremely lightweight application manager/launcher for the NigolOS operating system, with as little unnecessary baggage as possible. Application paths can be added to the launcher, and should a graphical environment be needed (and not currently in session) an xorg session can be launched for a given application.

Feetm

Feetm was a stock market simulation written in Python, and using MySQL as a database engine, which I worked on with a friend. One of my first forays into any remotely serious programming, it was arugably the single best learning experience I have had in computing to date, as not only did it prove instrumental in my learning python, it introduced me to server maintenance, database engines, many basic networking principles, and my first experience of linux outside of beginner friendly desktop distributions. While the web application was never finished, I often think back to this project as a means of reference for learning in programming, going into the deep end and 'figuring things out' through trial and error is one of the single best ways to actually learn programming concepts.
