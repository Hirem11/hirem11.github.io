NigolOS is a blanket term for a set of projects to develop an operating system, all of which fall into different 'versions' of the same essential principle. While I experimented with a few different options, there are only two iterations of NigolOS that have to date reached anything beyond mere tinkering, planning, or setup.

NigolOS the GNU/Linux Distribution

The less impressive, but considerably more complete (and for that matter practical) variant of NigolOS is the GNU/Linux operating system I compiled from scratch, and made reproducible as a tar.gz file. This operating system was primarily designed for use with two package managers (developed by a close friend) - kik, and kek - which allowed for installation of reunixance license programs from self hosted repositories. Other programs could easily be installed through included build tools, or through the configuration of a separate package manager.

NigolOS the standalone OS

The more impressive, but almost totally useless alternative was a 64 bit operating system written from scratch in C and Assembly. With a whopping 8 colours to draw from on a command line interface with rudimentary albeit function IO, the complete lack of drivers which inevitably comes with an individually developed operating system meant the Nigolos window manager port was dead on arrival.
